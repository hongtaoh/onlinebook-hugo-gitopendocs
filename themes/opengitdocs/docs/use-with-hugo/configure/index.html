Configure Hugo

Hugo expects a configuration file to build your site. OpenGitDocs includes one already set up; simply move it from the theme repository into your site root:

mv themes/opengitdocs/exampleSite/config.yaml .

Be careful not to overwrite an existing file named config.yaml, if that is not your intention.

Here’s the full config.yaml used to build this site (yes, the one you’re reading right now!):

baseURL: https://opengitdocs.com/hugo/
languageCode: en-us
title: OpenGitDocs for Hugo
theme: # Name of your theme, hopefully, opengitdocs
enableGitInfo: true # Show last modified date on pages
sectionPagesMenu: main

params:
  description: An open source static site solution for documentation websites.
  images:
  - /img/screenshot.png # Used in site metadata
  title: OpenGitDocs for Hugo
  logo: /img/book-open.svg # Path from root to your logo, best served 1:1 ratio
  ico: /img/book-open-dark.svg # Path to favicon
  colorTheme: "light" # "light" or "dark"
  mainColor: "" # Hex color for the sidebar, i.e. "#016087"
  sideBarExpanded: true # Show all subsections in sidebar on desktop screens
  footerText: "[OpenGitDocs on GitHub](https://github.com/opengitdocs/hugo)" # Shown at the bottom of pages

markup: # See https://gohugo.io/getting-started/configuration-markup for more
  highlight:
    codeFences: true
    guessSyntax: false
    hl_Lines: ""
    lineNoStart: 1
    lineNos: false
    lineNumbersInTable: false
    noClasses: true
    style: solarized-light
    tabWidth: 4

OpenGitDocs uses reasonable defaults for unset parameters where possible.

Hugo makes no assumptions, so if a necessary parameter is missing, you’ll see a warning when building or serving your site. Here are all configuration settings for Hugo configuration files.

Congratulations, you’re ready to preview and build your site!
